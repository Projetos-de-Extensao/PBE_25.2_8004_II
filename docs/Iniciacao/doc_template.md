## Introdução

<p align="justify"> O presente documento tem como objetivo apresentar a descrição e o detalhamento das etapas realizadas no desenvolvimento do projeto <b>PBE_25.2_8004_II</b>, que faz parte das atividades de extensão do curso de Engenharia de Software do IBMEC. O projeto visa o desenvolvimento de um sistema acadêmico web, integrando conceitos de engenharia de software, banco de dados, front-end e documentação técnica. Este documento busca registrar de forma organizada as metodologias aplicadas, os artefatos produzidos e as evoluções ocorridas durante as versões do projeto, servindo como base para futuras atualizações e como guia de consulta técnica. </p>
</p>

## Metodologia

<p align="justify"> A elaboração deste documento foi realizada de forma colaborativa pela equipe de desenvolvimento, seguindo uma abordagem incremental e iterativa. As etapas foram conduzidas a partir da coleta e análise de requisitos, seguida pela modelagem de casos de uso, diagramas de classes, protótipos de interface e implementação do sistema. Foram utilizadas as ferramentas <b>GitHub</b> para controle de versão, <b>Visual Studio Code</b> para desenvolvimento do código-fonte, <b>Draw.io</b> para modelagem dos diagramas UML e <b>Markdown</b> para estruturação da documentação. As atividades foram divididas entre os membros do grupo conforme as responsabilidades técnicas de cada um, garantindo o cumprimento dos prazos e a manutenção da padronização do código e da documentação. </p>

## Desenvolvimento 

Versão 1.0 — Estrutura Inicial do Projeto
<p align="justify"> Na primeira versão, foi definida a estrutura básica do projeto no GitHub, com a criação dos diretórios principais: <b>Frontend</b>, <b>Core</b>, e <b>Config</b>. O ambiente de desenvolvimento foi configurado para suportar as tecnologias necessárias, incluindo o uso de <b>Python/Django</b> no backend e <b>HTML, CSS e JavaScript</b> no frontend. Nessa fase, também foram criados os primeiros artefatos de documentação, como o diagrama de classes e o documento de casos de uso, que serviram como referência para o desenvolvimento dos módulos do sistema. </p>

Versão 1.1 — Desenvolvimento do Frontend
<p align="justify"> O foco da segunda versão foi a implementação da interface do usuário. Foram desenvolvidas as páginas de login para alunos e professores, bem como as telas de acesso ao sistema. As interfaces foram criadas utilizando <b>HTML5</b>, <b>CSS3</b> e <b>JavaScript</b>, com ênfase em design responsivo e usabilidade. O projeto também recebeu melhorias visuais com base nas cores institucionais do IBMEC, conforme orientações de design do grupo. Durante essa etapa, foram realizados testes locais para validar a navegação entre as páginas e o correto funcionamento dos botões de acesso e logout. </p>

Versão 1.2 — Diagramação e Documentação Técnica
<p align="justify"> Nessa versão, a equipe concentrou-se na elaboração e revisão dos diagramas UML, incluindo o <b>Diagrama de Classes</b> e o <b>Diagrama de Sequência</b> do caso de uso “Inscrição em Disciplina”. Os diagramas foram criados no Draw.io, exportados e adicionados à documentação do repositório, com o objetivo de facilitar o entendimento da estrutura do sistema e da interação entre os componentes. Além disso, foi criada a primeira versão deste documento de metodologia, consolidando as práticas adotadas e as ferramentas utilizadas. </p>


## Conclusão

<p align="justify"> A aplicação das técnicas de engenharia de software e o uso de ferramentas colaborativas, como o GitHub e o VS Code, foram essenciais para o sucesso do projeto <b>PBE_25.2_8004_II</b>. O registro detalhado das versões e o desenvolvimento incremental possibilitaram um maior controle sobre o progresso da equipe e a rastreabilidade das alterações realizadas. Este documento reflete o comprometimento do grupo com as boas práticas de documentação e serve como referência para trabalhos futuros e aprimoramentos no sistema. </p>
